IMPORTANT NOTICE FOR DEVELOPERS: this is a template that can only be used with jQuery/jQuery-ui/AXEL libraries sitting directly inside the same window (and not inside a parent window such as when opening inside an iframe) because of jQuery datepicker limitations. Thus for that reason it has been written as a self-transformed template with the required libraries included as you can see in its source code.

The "date" filter

The "date" filter turns a text entry field into a field for selecting a date using a calendar or the keyboard. It uses the jQuery UI Datepicker, hence it depends on jQuery and jQuery UI.

This filter is compatible with the "text" plugin.

Parameters

  • date_region (fr) : specifies the user interface language (currently only 'fr', but you can edit the filters/date.js source code to add another region)
  • date_format (ISO_8601) : specifies the date format used in XML output (read and write). You can use any datepicker date format string (e.g. 'dd/mm/yy'), or a $.datepicker constant name such as ISO_8601 (others are ATOM, COOKIE, RFC_822, RFC_850, RFC_1036, RFC_1123, RFC_2822, RSS, TICKS, TIMESTAMP, W3C)
  • minDate (undefined) : lower bound on the date that can be entered (in date_region format), the special value today is converted to the current day
  • maxDate (undefined) : upper bound on the date that can be entered (in date_region format), the special value today is converted to the current day
  • beforeShow (undefined) : to be documented

The special default value today will cause the default value to be replaced by the current day.

Javascript API

The filter creates a configure method which can be used to programmaticaly set the various parameters on a given plugin instance at runtime.
